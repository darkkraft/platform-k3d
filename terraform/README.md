# Terraform (OpenTofu) layout

Three composed roots, five reusable modules, zero logic in the stages. Roots
compose, modules do the work, `stages/<env>` injects pure data.

```
stages/<env>/{cluster,bootstrap,services}   # Terragrunt units: inputs + state only
        │ dependency (outputs: kube_context, cluster_id, …)
        ▼
terraform/cluster     Layer 0  k3d cluster + registry, image build/push (docker provider)
terraform/bootstrap   Layer 1  app namespace (PSA), Forgejo (seeded), ArgoCD root app, Vault gate
terraform/services    Layer 2  Vault KV + per-service creds + scoped ESO token (via ingress)
        │
        ▼
terraform/modules/    k3d-cluster · registry-images · forgejo · argocd · vault-config
```

Layer 0 deliberately has **no kubernetes/helm provider** — those need a
kube-context that only exists after the cluster does; bootstrap picks it up via
a Terragrunt `dependency`. The docker provider talks only to the local daemon,
so it lives in Layer 0 without that chicken-and-egg.

## File conventions (every root and module)

| File | Contents |
|---|---|
| `main.tf` | Layer header + the primary composition (kept small) |
| `<concern>.tf` | One concern per file: `forgejo.tf`, `argocd.tf`, `vault.tf`, `images.tf` |
| `variables.tf` | Every variable **typed + described**; inputs `validation{}`-checked (see Variable contract) |
| `locals.tf` | Naming stem (`<prefix>-<environment>`) + derived values |
| `outputs.tf` | Every output described |
| `providers.tf` / `versions.tf` | Provider configs / version constraints (pessimistic `~>`; exact versions locked in `.terraform.lock.hcl`) |
| `backend.tf` | Doc header only — the backend is generated by Terragrunt |

## Variable contract

- Error message style: `[${var.x}] x is not allowed: …` — the offending value is
  always in the message.
- Environments, ports, images, namespaces, durations, semver pins: all
  allow-listed or regex-validated **at the root AND re-validated inside modules**
  (modules are the reusable unit; they defend themselves).
- Cross-field checks where variables are objects: e.g. `cluster.http_port`,
  `https_port`, `api_port` must not collide.
- Policy parity: `image_tag = "latest"` fails at **plan time** for the same
  reason Kyverno's disallow-latest rejects it at admission.
- Grouped object variables (`cluster`, `registry`, `build`, `argocd`, `gitops`,
  `local_git`, `vault`, `eso`) with `optional()` defaults — stages override only
  what differs per environment.

## Imperative escape hatches (all of them)

Everything is provider-shaped except four steps that have no provider
equivalent, each kept inline, small, and idempotent:

1. `k3d cluster create` (modules/k3d-cluster) — k3d has no maintained provider; the CLI is the API.
2. `git push` seeding Forgejo (modules/forgejo/seed.tf) — no provider can push a git tree.
3. `forgejo-cli actions register` (modules/forgejo/runner.tf) — only reachable via `kubectl exec`.
4. Vault readiness gate (bootstrap/vault.tf) — waits for ArgoCD's wave-0 sync before Layer 2 runs.

## Enforcement

- `tofu fmt` + `tofu validate` (all three roots) + `tflint --recursive`
  (preset `all`: documented+typed variables, unused declarations, naming,
  version pins) run in CI as **hard gates** — see `.forgejo/workflows/ci-infra.yaml`.
- Pre-commit mirrors them (`.pre-commit-config.yaml`).

## Day-to-day

```bash
scripts/lazy.sh dev            # bring up an env end-to-end (cluster→bootstrap→services)
terragrunt run-all apply --terragrunt-working-dir stages/dev    # same, by hand
terragrunt apply --terragrunt-working-dir stages/dev/services   # one layer
scripts/lazy-down.sh dev       # teardown (cluster + state)
```
